# V2 模块设计

本文档对 `architecture-overview.md` 中定义的 V2 架构做稳定的模块级摘要，供后续实现使用。

## 核心正式模块

### 通政司
- 负责入口解析、校验与规范化。
- 只产出 `InboundEvent` 对象。
- 不负责编排、归档访问或模型执行。

### 中书省
- 负责主控策略、会话级路由和顶层推理流程。
- 决定直接回复、追问还是委派执行。
- 通过稳定契约与认知层、执行层交互。

### 翰林院 / 史馆
- 负责检索、记忆整理与历史上下文供给。
- 从归档与角色记忆中读取内容，为推理准备上下文。
- 长期记忆写入必须经过显式规则门控。

### 六部
- 负责执行 backend、工具注册表、容器生命周期与工件处理。
- host 与 container 执行都属于该层。
- `main` 中的旧实现只能作为设计参考，不能保留为正式 V2 代码。

### 门下省
- 负责回复校验与输出守卫。
- 对外发出的回复必须经过审批、过滤或拒绝。
- 它本身不生成内容。

## 支撑正式模块

### 数据 Schema
- 是所有正式 V2 模块共享的不可变契约层。
- 定义 `InboundEvent`、`ExecutionRequest`、`ExecutionResult`、`Reply` 等规范类型。

### 角色与资产
- 加载并校验 SOUL、ROLE、MEMORY、POLICY 等角色资产。
- 为主控层和执行层提供稳定的角色定义。

### 归档与状态
- 持久化任务记录、会话状态与执行输出。
- 为检索与审计提供结构化查询接口。

### 能力适配 / Provider / 配置 / 技能
- 通过显式内部契约隔离外部集成。
- 使 V2 能替换 provider 或能力来源，而不把外部框架假设泄露到整个工作区。

## 未来实现目录规则

当本分支开始正式实现时：

- `src/` 只放 V2 正式源代码。
- `tests/` 只放 V2 正式测试。
- `scripts/` 只放 V2 正式项目脚本。
- `main` 的旧代码不得进入这些目录。
